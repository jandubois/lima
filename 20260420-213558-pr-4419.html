Executive Summary ¶

Round 3 reviews the same commit (45823748) as Round 2 — no code has changed since 2026-03-29, so the previously identified gaps remain. The core implementation in TranslateHostPath is sound (longest-match on cleaned mount locations, filepath.Rel + filepath.IsLocal containment, hard errors), but mount locations and host paths are not symlink-resolved. On macOS this rejects legitimate paths whenever /tmp → /private/tmp, $HOME lives on an external volume, or any caller canonicalizes a path through readlink first. The bats instance now unconditionally mounts $HOME, which affects every other test that reuses that instance and creates host-side cleanup hazards on assertion failure. Finally, Glob and SearchFileContent still return raw guest paths that the agent cannot feed back into ReadFile/WriteFile because the new strict validation rejects them — pre-existing design, newly consequential.

Common failure cases on macOS:

  • /tmp is a symlink to /private/tmp. If the mount location is /tmp and the agent passes /private/tmp/foo, filepath.Rel("/tmp", "/private/tmp/foo") returns "../private/tmp/foo", IsLocal returns false, and the call fails with path "..." is not under any mounted directory.
  • $HOME on an external volume (e.g. /Users/jan → /Volumes/Data/Users/jan). The default template mounts ${HOME} as /Users/jan, so any caller that canonicalizes through the symlink target fails validation.

Every MCP filesystem and shell tool routes through TranslateHostPath, so this regression blocks read_file, write_file, list_directory, glob, search_file_content, and run_shell_command on affected paths. The new unit tests in toolset_test.go exercise only lexical parent/child cases; no test covers a symlinked prefix.

Fix: resolve symlinks on both the host path and each mount location before comparison. Tolerate EvalSymlinks errors (fall back to the cleaned path) so that write_file targeting a not-yet-existing file under a symlinked parent still works — resolve the deepest existing ancestor and re-append the unresolved suffix.

Design Observations ¶

Concerns ¶

  • Mount translation is duplicated across packages (in-scope) — pkg/mcp/toolset/toolset.go:121-145 reimplements host-to-guest translation already present in pkg/hostagent/inotify.go:95-109. The two copies have diverged: the hostagent version resolves symlinks, the toolset version does not. I1 is the first concrete bug caused by that divergence. Extracting a shared helper in a small mount-translation utility would unify symlink handling and give both call sites the same regression tests. Claude Codex Gemini

Strengths ¶

  • Containment via filepath.Rel + filepath.IsLocal (in-scope) — handles trailing slashes, .. components, same-prefix-different-directory cases (/home/user vs /home/user2), and Windows cross-drive paths with one primitive. Naive strings.HasPrefix would get all of these wrong. Claude Codex Gemini
  • Centralized enforcement (in-scope) — every MCP tool routes through TranslateHostPath rather than each tool repeating the policy. Claude Codex
  • Longest-match heuristic for overlapping mounts (in-scope) — after filepath.Clean, a longer canonical location string is always the more specific mount. Simple and correct. Claude
  • POSIX/native separator split (in-scope) — filepath.ToSlash(rel) + path.Join(*mount.MountPoint, ...) correctly produces a Linux path for the guest regardless of host OS. Claude

Testing Assessment ¶

Unit coverage is solid for lexical cases — empty, relative, exact match, subdirectory, overlapping mounts in both orders, nil MountPoint, trailing slashes, and Windows drive-letter paths via the absPath() helper.

Missing scenarios, ranked by risk:

  1. Symlinked host path or symlinked mount location — the primary failure mode on macOS (I1). A test with /private/tmp/foo against a /tmp mount would catch it.
  2. Case-insensitive filesystem match — S2.
  3. write_file creating a new file under a symlinked parent — relevant once I1 is fixed, since EvalSymlinks on a nonexistent target returns an error.
  4. Equal-length mount locations — S3.
  5. Windows cross-drive paths — filepath.Rel returns an error; no test exercises the err != nil branch.
  6. BATS integration on a host where $HOME is a symlink — probes I1 end-to-end.
  7. BATS end-to-end rejection of unmounted paths — no integration test asserts the new hard error through the MCP protocol.

Documentation Assessment ¶

The exported TranslateHostPath gains non-trivial behavior (mount containment + longest match) but no godoc describes the contract or the error cases. Callers in filesystem.go and shell.go propagate the error verbatim to the AI agent, so the error wording is part of the public contract and deserves a godoc comment.

The MCP tool schemas at pkg/mcp/msi/filesystem.go and pkg/mcp/msi/shell.go still describe the pre-PR contract (see S6); the AI agent consumes these strings and has no other way to learn the mount restriction.
