Executive Summary ¶

This PR replaces a broad len(inst.Config.Mounts) > 0 check in limactl shell with a precise mountsContainPath function that verifies the host working directory falls under a configured mount before generating a cd command in the guest. The fix eliminates spurious "No such file or directory" errors when only unrelated directories are mounted. Two issues require attention before merge: the home directory fallback is lost when the current working directory is outside mounts but home is mounted (a regression), and the path comparison omits symlink resolution (a gap the codebase's own patterns indicate is required).

2. Missing symlink resolution causes false negatives — cmd/limactl/shell.go:632-643 Claude Codex Gemini (important, gap) ¶
cmd/limactl/shell.go:632-643
func mountsContainPath(mounts []limatype.Mount, path string) bool {
	for _, m := range mounts {
		loc, err := localpathutil.Expand(m.Location)
		if err != nil {
			continue
		}
		if path == loc || strings.HasPrefix(path, loc+string(os.PathSeparator)) {
			return true
		}
	}
	return false
}

Neither path (from os.Getwd() at line 660) nor loc (from localpathutil.Expand, which calls filepath.Abs — no symlink resolution) are symlink-resolved. On macOS, /tmp is a symlink to /private/tmp. When os.Getwd() returns the resolved path /private/tmp/... but the mount location remains /tmp/lima, the prefix check at line 638 fails.

The codebase has an established pattern: pkg/hostagent/inotify.go:75 calls filepath.EvalSymlinks(m.Location) before comparing mount locations against host paths.

Fix: Resolve symlinks on both sides:

 func mountsContainPath(mounts []limatype.Mount, path string) bool {
+    if resolved, err := filepath.EvalSymlinks(path); err == nil {
+        path = resolved
+    }
     for _, m := range mounts {

         if err != nil {
             continue
         }
+        if resolved, err := filepath.EvalSymlinks(loc); err == nil {
+            loc = resolved
+        }

3. No unit tests for mountsContainPath — cmd/limactl/shell.go:632-643 Claude Codex (important, gap) ¶

The core logic of this PR has no test coverage. mountsContainPath has subtleties that warrant testing: exact directory match vs. prefix match, paths that share a prefix but diverge (e.g., /tmp/lima-foo vs. mount /tmp/lima), tilde locations, and Expand errors.

Fix: Add a table-driven test in shell_test.go:

func TestMountsContainPath(t *testing.T) {
    homeDir, err := os.UserHomeDir()
    assert.NilError(t, err)

    mounts := []limatype.Mount{
        {Location: "~"},
        {Location: "/tmp/lima"},
    }
    tests := []struct {
        name string
        path string
        want bool
    }{
        {"exact home", homeDir, true},
        {"subdir of home", filepath.Join(homeDir, "projects", "foo"), true},
        {"exact tmp mount", "/tmp/lima", true},
        {"subdir of tmp mount", "/tmp/lima/work", true},
        {"prefix but not subdir", "/tmp/lima-other", false},
        {"unrelated path", "/opt/project", false},
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            assert.Equal(t, mountsContainPath(mounts, tt.path), tt.want)
        })
    }

    t.Run("nil mounts", func(t *testing.T) {
        assert.Equal(t, mountsContainPath(nil, homeDir), false)
    })
}

Design Observations ¶

Concerns

  1. Path normalization logic is duplicated (in-scope) Codex Claude

    cmd/limactl/shell.go:632-638 implements its own mount/path matching rule, while pkg/hostagent/inotify.go:75-108 already has symlink-aware host-path translation logic. A shared helper in pkg/localpathutil that performs tilde expansion, absolute-path normalization, and symlink resolution would eliminate the "one caller remembered EvalSymlinks, another forgot" class of bugs.

Strengths

  1. The prefix check strings.HasPrefix(path, loc+string(os.PathSeparator)) at line 638 correctly prevents false positives from paths that share a prefix but are not subdirectories (e.g., /tmp/lima-other vs. /tmp/lima). Claude
  2. Replacing the len > 0 heuristic with an actual containment check is the right semantic fix. It resolves the FIXME comment and addresses the user-facing problem described in the PR. Claude Codex Gemini

Testing Assessment ¶

  1. mountsContainPath has zero test coverage — This is the core logic of the PR. The function handles prefix matching, tilde expansion, and error handling, all of which should be tested. (Highest risk.)
  2. No integration test for the behavioral change — The switch-case in shellAction now branches differently. The BATS tests in hack/bats/tests/ cover shell argument parsing, not the specific scenario where the CWD falls outside mounts.
  3. TestExpand is solid — Covers the main paths (tilde, absolute, error cases) and was updated to use t.TempDir() to fix the Windows CI failure flagged in review.

Documentation Assessment ¶
