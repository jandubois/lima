## Executive Summary

This PR replaces a broad `len(inst.Config.Mounts) > 0` check in `limactl shell` with a precise `mountsContainPath` function that verifies the host working directory falls under a configured mount before generating a `cd` command in the guest. The fix eliminates spurious "No such file or directory" errors when only unrelated directories are mounted. Two issues require attention before merge: the home directory fallback is lost when the current working directory is outside mounts but home is mounted (a regression), and the path comparison omits symlink resolution (a gap the codebase's own patterns indicate is required).

2. **Missing symlink resolution causes false negatives** — `cmd/limactl/shell.go:632-643` [Claude Opus 4.6, Codex GPT 5.4, Gemini 3.1 Pro] (important, gap)

```go

        loc, err := localpathutil.Expand(m.Location)
        if err != nil {
            continue
        }
        if path == loc || strings.HasPrefix(path, loc+string(os.PathSeparator)) {

```

Neither `path` (from `os.Getwd()` at line 660) nor `loc` (from `localpathutil.Expand`, which calls `filepath.Abs` — no symlink resolution) are symlink-resolved. On macOS, `/tmp` is a symlink to `/private/tmp`. When `os.Getwd()` returns the resolved path `/private/tmp/...` but the mount location remains `/tmp/lima`, the prefix check at line 638 fails.

The codebase has an established pattern: `pkg/hostagent/inotify.go:75` calls `filepath.EvalSymlinks(m.Location)` before comparing mount locations against host paths.

Fix: Resolve symlinks on both sides:

```diff

+    if resolved, err := filepath.EvalSymlinks(path); err == nil {
+        path = resolved
+    }
     for _, m := range mounts {
         loc, err := localpathutil.Expand(m.Location)
         if err != nil {
             continue
         }
+        if resolved, err := filepath.EvalSymlinks(loc); err == nil {
+            loc = resolved
+        }
         if path == loc || strings.HasPrefix(path, loc+string(os.PathSeparator)) {
             return true
         }
     }
     return false
 }
```

1. **Missing symlink resolution in mount comparison** — `cmd/limactl/shell.go:632-642` (important, gap)

```go

		loc, err := localpathutil.Expand(m.Location)
```

As highlighted in the repository conventions (and following the existing pattern in `pkg/hostagent/inotify.go:75-81`), mount locations in `lima` are `filepath.Clean`-ed but not symlink-resolved. If `m.Location` is a symlink (e.g., `/tmp/lima` pointing to `/private/tmp/lima`), and `path` is the fully resolved path returned by `os.Getwd()`, the literal string prefix check will fail. 

**Fix**: Evaluate symlinks for both the current path and the mount location before comparing them. This also incorporates the fix for the root directory trailing slash issue above:

```go
func mountsContainPath(mounts []limatype.Mount, path string) bool {
	evalPath, err := filepath.EvalSymlinks(path)
	if err != nil {
		evalPath = path // fallback
	}

	for _, m := range mounts {
		loc, err := localpathutil.Expand(m.Location)
		if err != nil {
			continue
		}
		
		evalLoc, err := filepath.EvalSymlinks(loc)
		if err != nil {
			evalLoc = loc // fallback
		}

		evalLocClean := filepath.Clean(evalLoc)
		if !strings.HasSuffix(evalLocClean, string(os.PathSeparator)) {
			evalLocClean += string(os.PathSeparator)
		}

		if evalPath == evalLoc || strings.HasPrefix(evalPath, evalLocClean) {
			return true
		}
	}
	return false
}
```
