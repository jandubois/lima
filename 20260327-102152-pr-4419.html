Executive Summary ¶

This PR replaces a TODO in the MCP toolset with mount validation logic: TranslateHostPath now verifies that a host path falls under a configured mount and translates it to the corresponding guest path. The core design (hard errors instead of warnings, filepath.Rel for prefix matching) is sound. The main issues are a first-match-wins bug with nested mounts, a filepath.Dir call on a POSIX guest path that breaks on Windows, and guest paths leaking from Glob/SearchFileContent results that cannot be fed back to other tools.

Critical Issues ¶

None.

Important Issues ¶

1. Nested mounts resolve to wrong guest path (important, regression) Codex ¶

The loop at line 124 returns the first matching mount, not the most specific one. Lima supports nested mounts (e.g., pkg/cidata/template_test.go:114-115 has both /Users/dummy and /Users/dummy/lima). With mounts ordered as ["/Users/dummy" → "/Users/dummy", "/Users/dummy/lima" → "/mnt/lima"], a host path /Users/dummy/lima/file.txt matches the parent mount first and translates to /Users/dummy/lima/file.txt instead of the correct /mnt/lima/file.txt. All MCP tool functions call TranslateHostPath, so every operation can target the wrong directory. The unit tests at toolset_test.go:95-108 only cover disjoint mounts.

pkg/mcp/toolset/toolset.go
func (ts *ToolSet) translateToGuestPath(hostPath string) (string, bool) {
	cleanPath := filepath.Clean(hostPath)
	for _, mount := range ts.inst.Config.Mounts {
		location := filepath.Clean(mount.Location)

		if cleanPath == location {
			return *mount.MountPoint, true
		}

		rel, err := filepath.Rel(location, cleanPath)
		if err == nil && !strings.HasPrefix(rel, "..") {
			rel = filepath.ToSlash(rel)
			guestPath := path.Join(*mount.MountPoint, rel)
			return guestPath, true
		}
	}

	return hostPath, false
}

Fix: Scan all mounts and select the longest matching mount.Location. Add a test with overlapping mounts in both parent-first and child-first order.

2. filepath.Dir on guest path breaks on Windows (important, gap) Claude ¶

guestPath is now always a POSIX path (e.g., /mnt/home-user/foo/bar.txt) because translateToGuestPath at line 134 constructs it with path.Join. On Windows, filepath.Dir converts forward slashes to backslashes, producing \mnt\home-user\foo, which SFTP would send to the Linux guest as an invalid path. The "path" package is already imported at line 11.

pkg/mcp/toolset/filesystem.go
	guestPath, err := ts.TranslateHostPath(args.Path)
	if err != nil {
		return nil, nil, err
	}
	dir := filepath.Dir(guestPath)
	err = ts.sftp.MkdirAll(dir)
	if err != nil {
		return nil, nil, err
	}

Fix:

-	dir := filepath.Dir(guestPath)
+	dir := path.Dir(guestPath)
3. HasPrefix(rel, "..") falsely rejects paths named ..something (important, regression) Gemini ¶

If a directory under a mount is named ..ssh (technically valid on Unix), filepath.Rel returns "..ssh" and strings.HasPrefix("..ssh", "..") is true, incorrectly rejecting a valid mounted path at line 132. The practical likelihood is very low — such directory names are rare — but the fix is simple. filepath.IsLocal (Go 1.20+) correctly handles this and also returns true for "." (exact mount match), removing the need for the separate cleanPath == location check at lines 127-129.

4. Glob and SearchFileContent return guest paths that fail re-entry (important, gap) Gemini ¶

Glob calls ts.sftp.Glob(pattern) at line 138 and returns absolute guest paths in the result.

pkg/mcp/toolset/filesystem.go
	guestPath, err := ts.TranslateHostPath(pathStr)
	if err != nil {
		return nil, nil, err
	}
	pattern := path.Join(guestPath, args.Pattern)
	matches, err := ts.sftp.Glob(pattern)
	if matches == nil {
		matches = []string{}
	}
	if err != nil {
		return nil, nil, err
	}

SearchFileContent runs git grep ... <guestPath> at line 178, which prefixes output lines with guest paths.

pkg/mcp/toolset/filesystem.go
	guestPath, err := ts.TranslateHostPath(pathStr)
	if err != nil {
		return nil, nil, err
	}
	if args.Include != nil && *args.Include != "" {
		guestPath = path.Join(guestPath, *args.Include)
	}
	cmdToolRes, cmdRes, err := ts.RunShellCommand(ctx, req, msi.RunShellCommandParams{
		Command:   []string{"git", "grep", "-n", "--no-index", args.Pattern, guestPath},
		Directory: pathStr, // Directory must be always set
	})

When the AI reads these results and passes them to ReadFile or WriteFile, TranslateHostPath rejects them because they are guest paths, not host paths. This works today only because the default mount maps $HOME → $HOME (same path), but breaks with any asymmetric mount. This is pre-existing design — before this PR, there was no translation at all — but the PR makes it more consequential by introducing strict validation. The fix is outside the scope of this PR, but worth tracking.

3. translateToGuestPath returns hostPath on false (suggestion, regression) Claude ¶

4. Nil MountPoint guard (suggestion, gap) Gemini ¶

MountPoint is *string and is dereferenced without a nil check at line 128 and line 134. Lima's defaults processing (limayaml.FillDefault) ensures MountPoint is always populated, but a continue guard would prevent panics from programmatic callers that bypass defaults.

Fix: Add inside the loop:

		if mount.MountPoint == nil {
			continue
		}

Design Observations ¶

Concerns ¶

  • Glob/SearchFileContent path round-tripping (in-scope) — Finding 4 describes a design gap where tool outputs contain guest paths that cannot be fed back as inputs. A proper fix requires translating guest paths back to host paths before returning results, or making all tools accept both host and guest paths. Worth addressing in a follow-up PR. Claude Gemini
  • Glob/SearchFileContent CWD dependency (future) — Both Glob (filesystem.go:126) and SearchFileContent (filesystem.go:163) fall back to os.Getwd() when no explicit path is provided. With mount validation active, the MCP server's working directory must be under a configured mount. Claude
  • Case-insensitive file systems (future) — translateToGuestPath uses exact string matching. On macOS (case-insensitive by default), /Users/Jan and /users/jan are the same directory, but the mount check would reject the non-matching case. Gemini

Strengths ¶

  • Correct separation of filepath for host-side operations and path for guest-side in translateToGuestPath (lines 123-134), with filepath.ToSlash bridging the two. Claude
  • The filepath.Rel approach correctly handles the /home/user2 vs. /home/user edge case without string manipulation, validated by the unit test at lines 80-92. Claude
  • Converting from soft warnings to hard errors is the right safety boundary for MCP tools. Claude Codex

Testing Assessment ¶

The unit tests cover core translation logic well. Missing scenarios ranked by risk:

  1. Nested/overlapping mounts — No test with mounts like /Users/dummy and /Users/dummy/lima in both orderings. Highest-risk gap given Finding 1.
  2. No mounts configured — Config.Mounts is empty. The code handles this correctly, but no test verifies it.
  3. Mount with nil MountPoint — Would panic at line 128/134. No test documents this implicit contract.
  4. Path with trailing slash — filepath.Clean strips trailing slashes, so it should work, but untested.
  5. BATS test for rejected unmounted path — No integration test verifies that requesting a path outside all mounts returns an error through the MCP protocol.

Documentation Assessment ¶

No documentation gaps. The change is internal to the MCP toolset and does not affect user-facing documentation.
