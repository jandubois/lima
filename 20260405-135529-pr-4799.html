Executive Summary ¶

This PR replaces the overly broad len(inst.Config.Mounts) > 0 check in limactl shell with a precise mountsContainPath function that verifies the host working directory falls under a configured mount before generating a cd command in the guest. Round 2 addresses three of five issues from the prior review: the home-directory fallback when CWD is outside mounts, symlink resolution via filepath.EvalSymlinks, and the root-mount double-separator edge case. Two prior issues remain open: mountsContainPath still has no unit tests, and the Windows non-WSL2 path format mismatch is unresolved (though masked by the WSL2 short-circuit for the common case).

I1. Windows non-WSL2 path format mismatch remains unresolved ¶
Codex GPT 5.4 Gemini 3.1 Pro (important, regression)

On Windows, hostCurrentDir (line 257) and hostHomeDir (line 264) are converted to guest-format paths (e.g., /c/Users/alice/project) via mountDirFromWindowsDir at lines 247 and 212/676. Mount locations in inst.Config.Mounts remain in native Windows format (e.g., C:\Users\alice). The string comparison in mountsContainPath (line 652) always fails because a Unix-style guest path never matches a Windows-style host path.

The || inst.VMType == limatype.WSL2 clause at line 257 masks this for WSL2 (the primary Windows backend), but the new home-directory fallback at line 264 lacks a WSL2 short-circuit:

cmd/limactl/shell.go:245-268
	hostHomeDir, err := os.UserHomeDir()
	if err == nil && runtime.GOOS == "windows" {
		hostHomeDir, err = mountDirFromWindowsDir(ctx, inst, hostHomeDir)
	}
	if err != nil {
		logrus.WithError(err).Warn("failed to get the home directory")
		hostHomeDir = ""
	}

	switch {
	case workDir != "":
		changeDirCmd = fmt.Sprintf("cd %s || exit 1", shellescape.Quote(workDir))
	case mountsContainPath(inst.Config.Mounts, hostCurrentDir) || inst.VMType == limatype.WSL2:
		changeDirCmd = fmt.Sprintf("cd %s", shellescape.Quote(hostCurrentDir))
		if hostHomeDir != "" {
			changeDirCmd = fmt.Sprintf("%s || cd %s", changeDirCmd, shellescape.Quote(hostHomeDir))
		}
	case syncHostWorkdir:
		changeDirCmd = fmt.Sprintf("cd %s", shellescape.Quote(destRsyncDir))
	case hostHomeDir != "" && mountsContainPath(inst.Config.Mounts, hostHomeDir):
		changeDirCmd = fmt.Sprintf("cd %s", shellescape.Quote(hostHomeDir))
	default:
		logrus.Debug("the host home does not seem mounted, so the guest shell will have a different cwd")
	}

On WSL2, the home-directory case (line 264) is unreachable because case 2 (line 257) always matches via || inst.VMType == limatype.WSL2. On a hypothetical non-WSL2 Windows VM (QEMU), both the CWD and home fallback branches would fail silently. Lima currently supports only WSL2 on Windows, so the practical impact is nil today.

This was issue #5 in the prior review and remains unaddressed.

Fix: Pass the unconverted native host path to mountsContainPath. Capture os.Getwd() and os.UserHomeDir() before Windows conversion and use those for mount matching, while continuing to use the converted paths for the cd commands.

This function contains four distinct behaviors — tilde expansion, symlink resolution, root-mount special case, and prefix matching with separator guard — yet has no tests. The PR adds tests for localpathutil.Expand (a dependency), but the core logic under review remains uncovered. This was issue #3 in the prior review and remains unaddressed.

cmd/limactl/shell.go:637-657
func mountsContainPath(mounts []limatype.Mount, path string) bool {
	resolvedPath, err := filepath.EvalSymlinks(path)
	if err != nil {
		resolvedPath = path
	}
	for _, m := range mounts {
		loc, err := localpathutil.Expand(m.Location)
		if err != nil {
			continue
		}
		resolvedLoc, err := filepath.EvalSymlinks(loc)
		if err != nil {
			resolvedLoc = loc
		}
		if resolvedPath == resolvedLoc ||
			resolvedLoc == string(os.PathSeparator) || strings.HasPrefix(resolvedPath, resolvedLoc+string(os.PathSeparator)) {
			return true
		}
	}
	return false
}

Fix: Add a TestMountsContainPath in cmd/limactl/shell_test.go (which already exists) with table-driven cases:

func TestMountsContainPath(t *testing.T) {
    root := t.TempDir()
    child := filepath.Join(root, "child")
    assert.NilError(t, os.Mkdir(child, 0o755))

    linkBase := t.TempDir()
    link := filepath.Join(linkBase, "root-link")
    assert.NilError(t, os.Symlink(root, link))

    tests := []struct {
        name   string
        mounts []limatype.Mount
        path   string
        want   bool

        {"exact match", []limatype.Mount{{Location: root}}, root, true},
        {"child path", []limatype.Mount{{Location: root}}, child, true},
        {"prefix but not subdir", []limatype.Mount{{Location: root}}, root + "-other", false},
        {"symlinked path", []limatype.Mount{{Location: root}}, filepath.Join(link, "child"), true},

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            assert.Equal(t, mountsContainPath(tt.mounts, tt.path), tt.want)
        })
    }

Design Observations ¶

Concerns ¶

  1. Path normalization split across multiple layers (future) Codex GPT 5.4 — hostCurrentDirectory() in cmd/limactl/shell.go:673, mount defaulting in pkg/limayaml/defaults.go:694, and mount setup in pkg/hostagent/mount.go:55 each normalize Windows paths at different times and in different ways. Centralizing "host path to shell-visible mount path" behind one helper would eliminate an entire class of mismatches like I1.
  2. changeDirCmd uses host paths as guest paths (future) Claude Opus 4.6 — The changeDirCmd at lines 258 and 265 uses host paths (hostCurrentDir, hostHomeDir) directly in guest shell commands. This works only because MountPoint defaults to equal Location (defaults.go:703). When a user sets a different MountPoint (e.g., host /Users/jan mounted at guest /home/jan), the cd targets the wrong path. Outside this PR's scope, but mountsContainPath already iterates mounts and could translate host paths to guest mount points.

Strengths ¶

  • The separator-guard pattern (resolvedLoc + string(os.PathSeparator)) prevents false prefix matches like /tmp/lima matching mount /tmp/lim. Claude Opus 4.6
  • The root-mount special case (resolvedLoc == string(os.PathSeparator)) cleanly avoids the // double-separator problem. Claude Opus 4.6 Gemini 3.1 Pro
  • The new home-directory fallback case at line 264 improves the user experience when the CWD is outside any mount but home is mounted. Claude Opus 4.6 Codex GPT 5.4
  • The symlink resolution pattern (resolve both sides, fall back to raw path on error) matches the established pattern in pkg/hostagent/inotify.go:75-81. Claude Opus 4.6 Codex GPT 5.4

Testing Assessment ¶

The PR adds solid tests for localpathutil.Expand (5 cases covering tilde, absolute, empty, and unsupported-tilde inputs). The core logic of this PR — mountsContainPath — remains untested. Untested scenarios ranked by risk:

  1. Prefix boundary correctness — /tmp/limactl must not match mount /tmp/lima (separator guard handles this, but no test guards against future refactors)
  2. Root mount — mount at / should match any absolute path
  3. Symlink resolution — path containing symlinks should match a mount at the resolved location
  4. Exact match — path exactly equals a mount location
  5. No matching mount — should return false, not panic on empty list
  6. Unexpandable mount location — mount with ~unknownuser/... should be skipped without crashing
