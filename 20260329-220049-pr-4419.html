Executive Summary ¶

This PR implements mount validation for MCP host paths, filling a long-standing TODO in TranslateHostPath. The approach — filepath.Rel + filepath.IsLocal with longest-match selection — is sound and well-tested. The main gap is that mount locations are matched lexically without symlink resolution, so paths like /private/tmp/foo fail to match a /tmp mount on macOS. The BATS test infrastructure needs minor cleanup resilience improvements.

1. Symlink aliases of mounted directories are rejected Codex ¶
pkg/mcp/toolset/toolset.go:121-139
func (ts *ToolSet) translateToGuestPath(hostPath string) (string, bool) {
	cleanPath := filepath.Clean(hostPath)
	bestGuestPath := ""
	bestLen := -1
	for _, mount := range ts.inst.Config.Mounts {
		if mount.MountPoint == nil {
			continue
		}
		location := filepath.Clean(mount.Location)
		rel, err := filepath.Rel(location, cleanPath)
		if err == nil && filepath.IsLocal(rel) {
			candidate := *mount.MountPoint
			if rel != "." {
				candidate = path.Join(candidate, filepath.ToSlash(rel))
			}
			if len(location) > bestLen {
				bestGuestPath, bestLen = candidate, len(location)
			}
		}
	}
}

The new validation matches hostPath against mount.Location after filepath.Clean, but Clean does not resolve symlinks. On macOS, /tmp is a symlink to /private/tmp and /var to /private/var. If a mount is configured with location /tmp (as Lima's defaults store it), a host path /private/tmp/foo will not match — filepath.Rel("/tmp", "/private/tmp/foo") yields "../../private/tmp/foo", which filepath.IsLocal rejects.

The codebase already handles this exact problem for inotify watchers:

pkg/hostagent/inotify.go:70-93
func (a *HostAgent) setupWatchers(events chan notify.EventInfo) error {
	for _, m := range a.instConfig.Mounts {
		if !*m.Writable {
			continue
		}
		symlink, err := filepath.EvalSymlinks(m.Location)
		if err != nil {
			return err
		}
		if m.Location != symlink {
			mountSymlinks[symlink] = m.Location
		}
		if m.MountPoint != nil && m.Location != *m.MountPoint {
			mountLocations[m.Location] = *m.MountPoint
		}

		logrus.Infof("enable inotify for writable mount: %s", m.Location)
		err = notify.Watch(path.Join(m.Location, "..."), events, GetNotifyEvent())
		if err != nil {
			return err
		}
	}
	return nil
}

2. No nil-instance guard on public TranslateHostPath Claude ¶
pkg/mcp/toolset/toolset.go:106-118
func (ts *ToolSet) TranslateHostPath(hostPath string) (guestPath string, err error) {
	if hostPath == "" {
		return "", errors.New("path is empty")
	}
	if !filepath.IsAbs(hostPath) {
		return "", fmt.Errorf("expected an absolute path, got a relative path: %q", hostPath)
	}

	guestPath, isMounted := ts.translateToGuestPath(hostPath)
	if !isMounted {
		return "", fmt.Errorf("path %q is not under any mounted directory", hostPath)
	}
	return guestPath, nil
}

translateToGuestPath dereferences ts.inst.Config.Mounts at line 125 without a nil check. All current callers guard against ts.inst == nil before calling TranslateHostPath, so this is safe in practice. But the public method has no precondition documented.

Fix: either add if ts.inst == nil { return "", errors.New("instance not registered") } at the top of TranslateHostPath, or document the precondition in a comment.

Design Observations ¶

Concerns ¶

(in-scope) Mount-prefix resolution is now security-critical but performed ad hoc on every call. Moving canonical mount preparation (symlink resolution, platform normalization) into RegisterInstance would make the validation explicit shared state rather than re-derived per call, and would eliminate the per-call EvalSymlinks overhead. Codex

Strengths ¶

  • Centralizing validation in toolset.go:106 protects every MCP filesystem and shell entrypoint instead of relying on scattered per-tool checks. Claude Codex
  • Using filepath.Rel + filepath.IsLocal for containment checking correctly handles traversal attacks, similar-prefix non-matches (/home/user2 vs /home/user), and Windows cross-drive paths — all without manual string prefix matching. Claude
  • The longest-match heuristic (len(location) > bestLen) is simple and correct: after filepath.Clean, a longer canonical path string is always a more specific mount. Claude
  • Switching filepath.Dir to path.Dir for the guest path in filesystem.go:90 correctly uses POSIX path semantics for what is now a Linux path. Claude Codex

Testing Assessment ¶

Unit tests are comprehensive, covering overlapping mounts, exact matches, similar prefixes, nil MountPoint, empty mounts, trailing slashes, relative and empty paths, and Windows paths via the absPath() helper.

Untested scenarios ranked by risk:

  1. Symlink aliases of mounted host paths — the highest-risk gap. A test with /private/tmp/foo against a /tmp mount would catch the symlink issue on macOS.
  2. BATS end-to-end rejection of unmounted paths — no integration test asserts the new hard error for an unmounted path on any MCP tool.
  3. Nil MountPoint with empty string — *mount.MountPoint == "" would produce a relative guest path via path.Join("", rel).
  4. Symlink in host path itself — filepath.Rel operates lexically; a host path containing a symlink that escapes a mount would pass validation but resolve to an unmounted location in the guest.

Documentation Assessment ¶

The user-facing MCP schema text (pkg/mcp/msi/filesystem.go, pkg/mcp/msi/shell.go) needs updating to describe the "mounted host path only" requirement. No other documentation changes are required.

Commit Structure ¶

Single squashed commit as requested by reviewer AkihiroSuda. The commit message "added validation for hostPath" is terse — it would benefit from stating what was validated (mount containment) and why (security boundary for MCP tools).
